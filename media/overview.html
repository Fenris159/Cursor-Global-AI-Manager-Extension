Cursor Global AI Manager

This extension lets you manage user-level Cursor AI settings: rules, skills, subagents, commands, and hooks that apply to every workspace you open, not just the current project. You edit files in your global ~/.cursor/ folder from one place.

How to open this panel: Click Manage User AI in the status bar (bottom of the window), or open the Command Palette (Ctrl+Shift+P / Cmd+Shift+P) and run Manage User AI.

Changes you make here do not apply to existing chats. To use new or updated rules or skills, start a new chat or right-click the agent in the sidebar and choose Fork chat.

Global impact. Changes you make here apply to every Cursor workspace (every project and folder). Add or edit content only when you intend it to affect all projects. Consider testing in a single project first if unsure.

What's in the sidebar

  • #README — This overview (you are here).
  • Rules — Instructions and context the AI follows in every project (e.g. coding style, project conventions).
  • Skills — Reusable agent skills (custom behaviors or workflows) stored as folders with a SKILL.md.
  • Subagents — Custom agent configurations (e.g. specialized task agents). Files live in ~/.cursor/agents/ (Cursor’s folder name).
  • Commands — Custom slash commands you can run from the chat input.
  • Hooks — Scripts that run when the agent does certain things (e.g. after editing a file, before running a shell command).

How to use

Expand a category in the sidebar (Rules, Skills, Subagents, Commands, or Hooks), then select a file to preview it. Use Edit to open the file in the editor. Use + Create new to add a new rule, skill, subagent, or command. For a skill folder, use the pencil icon to open that skill in a new window. Use the Import button (folder icon) in each category header to add files or skill folders from disk. Use Export (document-with-arrow) on a file or skill folder to save a copy elsewhere. Use Sync to Workspace (sync icon) to copy the selected item into this project's .cursor folder so you can share it with the team (e.g. for developers who don't have it in their global ~/.cursor/).

Rules format. In Cursor's native Settings → Rules, a rule with only alwaysApply: false and no description or globs may show "Incorrect format". That's expected: when a rule isn't always applied, Cursor needs either a description (Apply Intelligently) or globs (Apply to Specific Files). Without either, the rule only applies when you @mention it in chat (Apply Manually).
Settings → Rules "User" tab. Rules you manage here (in ~/.cursor/rules/) may not appear under the User filter in Cursor's native Settings → Rules. That's a Cursor UI quirk: the "User" tab often shows only rules stored in Cursor's internal state. Your file-based rules still apply globally and will appear under All (and in project filters). They are loaded and used by the AI.

Import, Export, and Sync

  • Import — Add files or skill folders from your machine into a category. Rules are normalized to valid frontmatter.
  • Export — Save a copy of a file or skill folder to any location you choose.
  • Sync to Workspace — Copy the selected global file or skill folder into .cursor/ in the open workspace. Creates the folder if needed. Requires a folder to be open (File → Open Folder).
Backups. Before any overwrite (Save, Import, or Create new with an existing name), the extension copies the current file or skill folder to ~/.cursor/.backups/ with a timestamped name (e.g. 20250302T143045-rules-my-rule.mdc or 20250302T143045-skills-my-skill/). To restore, copy a backup from .backups/ back into the matching category folder under ~/.cursor/.

Finding the user-level .cursor folder

The user-level .cursor folder (rules, skills, agents, commands, hooks) lives in your home directory. Subagents are stored in agents/, not “subagents”. Paths vary by OS:

  • Windows — %USERPROFILE%\.cursor (e.g. C:\Users\YourName\.cursor)
  • macOS — ~/.cursor (e.g. /Users/YourName/.cursor)
  • Linux — ~/.cursor (e.g. /home/YourName/.cursor)

To open it quickly: run File → Open Folder and paste the path above for your OS, or use your terminal (cd ~/.cursor on macOS/Linux, cd %USERPROFILE%\.cursor on Windows).

Portability and syncing across machines

Cursor does not offer a built-in portable mode. To keep your user-level AI config (rules, skills, subagents, commands) consistent across machines or backups:

Custom data directory. You can launch Cursor with a custom data directory using --user-data-dir (e.g. cursor --user-data-dir=/path/to/my-config). This changes where Cursor stores app settings, extensions, and session state. Note: some user-level features (e.g. agent discovery) have been reported to ignore this flag and still use ~/.cursor. For reliable portability of rules, skills, subagents, and commands, prefer syncing the ~/.cursor folder itself (see options below).
  • Copy the .cursor folder — Zip or copy ~/.cursor to another machine and place it in the same home-directory path. Overwrites existing content.
  • Version control — Put ~/.cursor in a private Git repo and clone it on each machine. Commit after changes; pull before starting work.
  • Cloud storage — Sync ~/.cursor via Dropbox, Google Drive, or similar. Use a dedicated subfolder (e.g. CursorConfig) and symlink it to ~/.cursor on each machine if you prefer not to sync the whole home directory.
  • Export / Sync to Workspace — Use this extension's Export or Sync to Workspace to move specific rules or skills into a project repo. Others can then copy them into their own ~/.cursor/ or use them from the project's .cursor/ folder.
Cursor app config vs. user AI config. Cursor itself stores app settings (e.g. themes, keybindings) in platform-specific locations: %APPDATA%\Cursor (Windows), ~/Library/Application Support/Cursor (macOS), ~/.config/Cursor (Linux). The ~/.cursor folder holds only user-level AI content (rules, skills, subagents, commands, hooks) that this extension manages.
